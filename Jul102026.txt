As part of enabling Azure Backup for our virtual machines via the Recovery Services vault, we need to finalize the backup policy type — Azure offers two options: Standard and Enhanced. Since this decision affects cost, flexibility, and future backup capabilities, I wanted to lay out a comparison to support your decision.

Our current requirement is straightforward: daily backups, retaining the last 30 days of recovery points for each VM. Both policy types can technically satisfy this, but there are longer-term differences worth considering.

COMPARISON: STANDARD vs ENHANCED POLICY

1. Backup Frequency
   - Standard: Daily or Weekly only
   - Enhanced: Hourly (every 4/6/8/12/24 hrs), Daily, or Weekly — useful if RPO requirements tighten in the future

2. Retention & Recovery Points
   - Standard: Standard daily/weekly/monthly/yearly retention tiers
   - Enhanced: Same retention tiers, plus configurable instant-restore snapshot retention (1–30 days) for faster recoveries without pulling from the vault tier

3. Resiliency
   - Standard: Snapshot-tier recovery points are not zone-resilient
   - Enhanced: Snapshot-tier recovery points ARE zone-resilient — better protection against zonal failures

4. VM/Disk Compatibility
   - Standard: Limited support for newer VM configurations (e.g., Trusted Launch VMs require workarounds; no Premium SSD v2/Ultra Disk support)
   - Enhanced: Native support for Trusted Launch VMs, Premium SSD v2, and Ultra Disks; supports multi-disk crash-consistent snapshots and selective disk exclusion (backup only the disks that matter, reducing cost)

5. Network Isolation
   - Standard: Limited support for VMs with public network access disabled
   - Enhanced: Full support for VMs with public network access disabled (private endpoint scenarios) — relevant if we tighten network security on these VMs later

6. Cost
   - Standard: Generally lower cost, since fewer recovery points/snapshots are generated by default
   - Enhanced: Slightly higher snapshot-related cost due to additional recovery points, though selective disk backup can offset this

7. Migration Path
   - Standard → Enhanced: Supported (one-directional)
   - Enhanced → Standard: NOT supported — once a VM is moved to Enhanced, it cannot be reverted to Standard

RECOMMENDATION

For our immediate requirement (daily backups, 30-day retention), the Standard policy fully meets the need and is the lower-cost option. However, given that migration only works one-way (Standard to Enhanced, not back), and that Enhanced adds meaningful future-proofing — zone resiliency, Trusted Launch/Premium SSD v2 support, and compatibility with network-isolated VMs — there's a case for adopting Enhanced now if we anticipate any of the following in the next 12–24 months:
- Stricter RPOs requiring more frequent backups
- Migrating VMs to Trusted Launch or newer disk types
- Disabling public network access on these VMs
- Stronger resiliency requirements (zone-level protection)

If none of the above are on our roadmap, Standard is the more cost-efficient choice today.
